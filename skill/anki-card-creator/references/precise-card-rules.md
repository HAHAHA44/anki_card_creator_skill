# Precise Card Rules

These rules are derived from Soren Bjornstad's article:

https://controlaltbackspace.org/precise/

Use them as drafting constraints for the Anki deck spec.

## Rules

1. One card should test one thing.
2. Each prompt should have one intended answer.
3. Avoid prompts asking for arbitrary examples.
4. Avoid prompts asking the learner to enumerate a whole list.
5. Avoid yes-no prompts when a more informative prompt is possible.
6. Make the prompt understandable without outside context.
7. Rewrite or split ambiguous cards before packaging.

## Rewrite Patterns

### Bad: Example Prompt

```text
What is an example of a non-combinatorial circuit?
```

### Better: Recognition Prompt

```text
Memory is an example of what kind of circuit?
```

### Bad: Vague Prompt

```text
Why is the Earth's rotation slowing down?
```

### Better: Precise Prompt

```text
What gravitational effect causes the Earth's rotation to slow over time?
```

## Packaging Rule

If a card draft is ambiguous, over-broad, list-like, or answerable in multiple equally valid ways, revise the card before calling the MCP.
